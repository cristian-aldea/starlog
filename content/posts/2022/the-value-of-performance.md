As software developers, we should strive to solve problems, not create more of them.

Recently, I decided to rewrite many of my webapps to use ViteJS and vanilla TypeScript, and rewrote the entire backend for [Vote-It](TODO) from NodeJS to Golang.

I did this for many reasons, some of which I'll cover in my blog posts. Performance is the reason I will explore in this post.

# User Experience

The most obvious reason why software developers chase performance is for the user experience. A tool isn't very useful if it's slow to use and takes forever to get anything done.

Having efficient algorithms, using a powerful and fast language can leads to programs which offer a great user experience.

# Cost

If your program is slow, it'll require better more memory and CPU cycles as a better, faster program.

# Accessibility

One aspect that programmers often forget is accessibility. Not every user will be running the latest hardware, and not everyone will have unlimited bandwidth to download your application.

In web, one of the key metrics for performance is the size of your Javascript bundle
